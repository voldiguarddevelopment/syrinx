# Syrinx LM Parity Contract

This document is the **exact** specification a Rust implementation must replicate to
match the golden fixtures in `golden/*.json`. The fixtures were generated by
`reference.py` (pure-Python, stdlib only). An implementer who reads **only this file**
must be able to reproduce every golden within its stated tolerance.

All math is **f32** (IEEE-754 single precision). Tolerances: `1e-4` for single ops,
`1e-3` for the full LM forward. Comparison is **max-abs** elementwise: a fixture passes
iff `max_i |got_i - golden_i| <= tol`.

Tensors are row-major. Shapes use `[outer, ..., inner]`.

---

## 1. Golden JSON format

Each fixture is one JSON object:

```json
{ "name": str, "input": {...}, "shape": [int,...], "data": [float,...], "tol": float }
```

- `data` is the **flattened, row-major** output (except `causal_mask`, see §4.9).
- `shape` gives how to re-fold `data`.
- `input` records the exact input that produced the output (also given below).

---

## 2. PRNG + weight generation (§Weights)

Weights are **not** stored in a file. Every named tensor's values are generated
deterministically from its name. Replicate this exactly.

### 2.1 Name hash — FNV-1a 64-bit (unsalted, stable)

```
offset_basis = 0xCBF29CE484222325
prime        = 0x00000100000001B3
h = offset_basis
for each byte b of name.utf8_bytes():     # UTF-8 of the tensor name string
    h = h XOR b                            # XOR low 8 bits
    h = (h * prime) mod 2**64
seed = h
```

(Do **not** use a language's built-in/​salted hash. Use exactly this FNV-1a.)

### 2.2 xorshift64 stream

State is `u64`. If `seed == 0`, substitute `seed = 0x9E3779B97F4A7C15` (cannot occur
for the names used here, but guard it). Each draw advances the state **then** emits it:

```
x = state
x ^= (x << 13)        # all shifts/mults wrap mod 2**64
x ^= (x >> 7)
x ^= (x << 17)
state = x
emit x                # the emitted value is the post-update state
```

### 2.3 u64 -> f32 weight value

For each emitted `u64 x`:

```
u        = (x >> 11) * (1.0 / 2**53)     # f64 in [0, 1)  (top 53 bits)
centered = u * 2.0 - 1.0                  # in (-1, 1)
value    = f32(centered * 0.02)           # scale to ~0.02 magnitude
```

So `weights(name, count)` = the first `count` such values from the stream seeded by
`fnv1a_64(name)`. A flat weight list fills a matrix **row-major**: for `W[out, in]`,
element index `r*in + c` is `W[r][c]`.

**Golden:** `weights_sample.json` = `weights("tok_embeddings", 8)`.

## 3. Tensor-name → weight mapping (LM)

Config: `vocab=512, dim=128, n_layers=4, n_heads=4, head_dim=32, ffn_hidden=256,`
`rope_theta=10000, eps=1e-5`. Note `head_dim = dim / n_heads = 32`.

| Tensor name | Shape | Role |
|---|---|---|
| `tok_embeddings` | `[vocab, dim]` | token embedding table (row = token id) |
| `layers.{L}.attention_norm.weight` | `[dim]` | pre-attn RMSNorm weight |
| `layers.{L}.attention.wq.weight` | `[dim, dim]` | Q proj (no bias) |
| `layers.{L}.attention.wk.weight` | `[dim, dim]` | K proj (no bias) |
| `layers.{L}.attention.wv.weight` | `[dim, dim]` | V proj (no bias) |
| `layers.{L}.attention.wo.weight` | `[dim, dim]` | output proj (no bias) |
| `layers.{L}.ffn_norm.weight` | `[dim]` | pre-FFN RMSNorm weight |
| `layers.{L}.feed_forward.w1.weight` | `[ffn_hidden, dim]` | SwiGLU **gate** |
| `layers.{L}.feed_forward.w3.weight` | `[ffn_hidden, dim]` | SwiGLU **up** |
| `layers.{L}.feed_forward.w2.weight` | `[dim, ffn_hidden]` | SwiGLU **down** |
| `norm.weight` | `[dim]` | final RMSNorm weight |
| `output.weight` | `[vocab, dim]` | lm_head — **separate / untied** |

`{L}` ranges `0..n_layers-1`. All linear projections are **bias-free**. Names are the
literal strings above (e.g. `layers.0.attention.wq.weight`).

## 4. Core ops — exact formulas

All `linear`/`matmul` accumulation may be done in f32 or f64; the reference accumulates
in f64 and rounds each output cell to f32. Either stays within tolerance.

### 4.1 matmul(A[m,k], B[k,n]) -> [m,n]
`out[i][j] = f32( sum_{p=0..k-1} A[i][p] * B[p][j] )`. Sum over the inner dim `k`.
Golden `matmul.json`: A=`[[1,2,3],[4,5,6],[7,8,9]]`, B=`[[9,8,7],[6,5,4],[3,2,1]]`.

### 4.2 add(A, B) / mul(A, B)
Elementwise, equal shapes: `f32(a + b)` / `f32(a * b)`.

### 4.3 linear(x[*, in], W[out, in], b[out]?) -> [*, out]
`W` is stored `[out, in]` (PyTorch `nn.Linear.weight` convention: row `o` = output
neuron `o`'s weights). `y[i][o] = f32( sum_{k} x[i][k] * W[o][k] (+ b[o]) )`.
Golden `linear.json` includes a bias; LM linears pass **no** bias.

### 4.4 rmsnorm(x[*, d], w[d], eps=1e-5)
```
ms      = (1/d) * sum_k x[k]^2          # mean of squares over last axis
inv     = 1 / sqrt(ms + eps)            # eps INSIDE the sqrt
y[k]    = f32( (x[k] * inv) * w[k] )
```
`eps = 1e-5` exactly, added **inside** the sqrt. Golden `rmsnorm.json`.

### 4.5 softmax(x) — over the last axis
```
m       = max_k x[k]
e[k]    = exp(x[k] - m)                  # subtract row max for stability
s       = sum_k e[k]
y[k]    = f32( e[k] / s )
```
Golden `softmax.json`.

### 4.6 silu(x) — elementwise
`silu(v) = f32( v * sigmoid(v) )`, `sigmoid(v) = 1 / (1 + exp(-v))`.
Golden `silu.json`.

### 4.7 rope(x[T, n_heads, head_dim], positions[T], theta=10000)
Pairing is **adjacent / interleaved**: dims `(2i, 2i+1)` rotate together,
`i = 0 .. head_dim/2 - 1`.
```
inv_freq[i] = theta ** ( -(2*i) / head_dim )
angle       = pos * inv_freq[i]
c = cos(angle); s = sin(angle)
a = x[2i]; b = x[2i+1]
x'[2i]   = f32( a*c - b*s )
x'[2i+1] = f32( a*s + b*c )
```
At `pos = 0` RoPE is the identity (cos=1, sin=0). Golden `rope.json`:
`x=[[[1,0,0.5,-0.5]],[[0.3,0.7,-0.2,0.9]]]`, positions `[0,1]`, theta `10000`.

### 4.8 embed(ids, table[V, d])
`out[i] = copy of table[ids[i]]`. Golden `embed.json`.

### 4.9 causal_mask(T) -> [T, T] (additive)
`mask[i][j] = 0.0 if j <= i else -inf`. Golden `causal_mask.json` encodes `-inf` as the
JSON **string** `"-inf"` (so `data` there is `[[float|"-inf"]]`, not flattened). When
added to scores, `-inf` drives `exp()` to 0 after the softmax max-subtraction.

## 5. LM forward — exact block order

`forward(token_ids[T]) -> logits[T, vocab]`. `positions = [0, 1, ..., T-1]`.

```
h = embed(token_ids, tok_embeddings)               # [T, dim]
for L in 0..n_layers-1:
    h = block(h, L, positions)
h = rmsnorm(h, norm.weight, eps)                   # final norm
logits = linear(h, output.weight)                  # [T, vocab], no bias
```

### 5.1 Block (pre-RMSNorm residual order)
```
# attention sub-block
n1   = rmsnorm(h, layers.L.attention_norm.weight, eps)
h    = h + attention(n1, L, positions)             # residual on PRE-norm h
# ffn sub-block
n2   = rmsnorm(h, layers.L.ffn_norm.weight, eps)
h    = h + swiglu_ffn(n2, L)                        # residual on PRE-norm h
```
Both residuals add the sub-block output to the **input** `h` (the value before that
sub-block's norm), not to the normed tensor.

### 5.2 attention(x[T,dim], L, positions) -> [T,dim]
```
q = linear(x, wq); k = linear(x, wk); v = linear(x, wv)     # [T, dim], no bias
# reshape [T, dim] -> [T, n_heads, head_dim]: head hi takes columns
#   [hi*head_dim : (hi+1)*head_dim]  (contiguous split, head-major-within-row)
q = rope(q, positions); k = rope(k, positions)               # v is NOT roped
scale = 1 / sqrt(head_dim)                                    # = 1/sqrt(32)
for each head hi:
    scores[i][j] = f32( (q[i,hi] · k[j,hi]) * scale + causal_mask[i][j] )
    probs        = softmax(scores)                            # over j (keys)
    ctx[i, hi*head_dim + d] = f32( sum_j probs[i][j] * v[j, hi, d] )
out = linear(ctx, wo)                                         # [T, dim], no bias
```
- Scaling `1/sqrt(head_dim)` is applied to the **dot product before** the mask add.
- RoPE applies to Q and K only (not V).
- Head split is contiguous: head `hi` owns columns `hi*head_dim .. (hi+1)*head_dim-1`.

### 5.3 swiglu_ffn(x[T,dim], L) -> [T,dim]
```
gate = silu( linear(x, w1) )          # [T, ffn_hidden]   (w1 = gate)
up   = linear(x, w3)                  # [T, ffn_hidden]   (w3 = up)
out  = linear( gate * up , w2 )       # [T, dim]          (w2 = down)
```
SwiGLU = `down( silu(x·w1) ⊙ (x·w3) )`. Elementwise `⊙` on the hidden dim.

**Golden:** `lm_forward.json` = `forward([1, 5, 9, 2, 0])` -> shape `[5, 512]`,
tol `1e-3`.

---

## 6. Determinism

Re-running `python3 reference.py` regenerates byte-identical JSON. A Rust port matching
this contract reproduces every `data` array within its `tol`. There is **no** randomness
in the LM forward — weights are a pure function of tensor names via §2.
